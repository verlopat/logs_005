# Blockchain-Enabled Tamper-Proof Security Event Logger

This repository implements **Objective 2** of the research project: a production-style, permissioned blockchain logging architecture for cloud anomaly detection events. It uses **Hyperledger Fabric 2.x** for immutable on-chain anchoring, **IPFS** for full off-chain payload storage, and **ECDSA/X.509** signatures for non-repudiation.

## Features

- Hyperledger Fabric 2.x private network with 1 orderer, 2 peers, CouchDB, Fabric CA
- Go chaincode with `LogSecurityEvent`, `VerifyEvent`, `QueryEventHistory`
- IPFS-backed off-chain storage; only SHA-256 hash + metadata on-chain
- Python SDK integration layer for submit/query flows
- ECDSA signing with Fabric CA-issued identities
- Compliance reporting (ISO 27001 / SOC 2 / NIST SP 800-92 style exports)
- Docker Compose based local bring-up

## Run

### 1. Bootstrap the Fabric network

```bash
python3 Main.py bootstrap
```

### 2. Check health

```bash
python3 Main.py health
```

### 3. Log a sample event end-to-end

```bash
python3 Main.py demo-log
```

### 4. Query audit history

```bash
python3 Main.py history
python3 Main.py history vm-prod-001
```

### 5. Verify a specific event

```bash
python3 Main.py verify <event_id>
```

### 6. Generate a compliance report

```bash
python3 Main.py report ISO27001
python3 Main.py report SOC2 vm-prod-001
```

Generated reports are written to `output/reports/`.

## Notes

- The current implementation is designed as an MVP / research-grade prototype for Objective 2.
- `fabric-sdk-py` can be brittle depending on local gRPC/protobuf versions; pinning is included in `requirements.txt`.
- The network profile expects crypto material generated by `bootstrap.sh`.
- For production, move to Kubernetes, external CA management, secret storage, and audited chaincode CI/CD.
